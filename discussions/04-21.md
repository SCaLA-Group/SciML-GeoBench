## Datasets

* SPE11: Well specified mathematical model, relatively rich dataset, 34 submissions, 200 snapshots of 8 variables each
* FluidFlower: Two experimental setups, different in geology, relatively scarce, half-hoursnapshots, 200x400 relatively coarse resolution

## Questions to ML

* FluidFlower: Is it possible to utilize the dataset from one setup to predict the other setup?
* SPE11: Can we speed up simulations? Not well suited for benchmarking.

## Comments

### General
* Would need more setups and datasets. Need a hierarchical approach. Ask for more datasets from other sources.
* For both SPE11 and FluidFlower, further simulation results could be produced if needed.

### FluidFlower
* Possibly work on patches rather than the whole images. Do we know which ML methods would work? Maybe learn from weather forecast?
* History matching FluidFlower took many thousands simulations. Code would still be available.
* Use a calibrated simulation model as ground truth.

### SPE11
* Should we just train on one model?
* Can we learn from the dataset to address uncertainty?
* Try to derive distributions reflecting the variability in the results.
* Decompose system into smaller units.
